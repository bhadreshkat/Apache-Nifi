What is Apache Nifi?

  • Next

Apache Nifi is a powerful, open-source data flow management system that automates the flow of data between software systems. It provides a user-friendly, web-based interface that allows users to easily create, control, and monitor data flows through a drag-and-drop interface.

How to configure Nifi?

  • Next

Nifi is very easy to use and configure. The Nifi UI is pretty basic and understandable, which makes Nifi very fun and easily understandable and easy to use for any beginner. Let's understand the use of tools in Nifi.

  • Processor
  • Input Port
  • Output Port
  • Process Group
  • Remote Process Group
  • Funnel
  • Import from Registry
  • Label

All the icons that are shown here is used to access and configurate Nifi in our own manner of use. The Processor is used to add new processor into the hierarchy and can be used to get data, store data and mmany more functions. The input/output port is used to specify the flow of data, i.e from which processor the data is flowing and where it is going. The Process group is used to group the process into one, after that any changes mmade into any processor in the group, it applies to the rest of the processors in that group. The same function is of Remote Process Group, It just adds the functionality of using and accessing that group remotely. Going forward, Funnel is used to take input of data from multiple processor and dumping the results into a single processor, it works same as its name. The Import from registry is used to import the customs instructions from our registry. And lastly the Label is used to label the processor individually for our clarification of use.

More on how to configure Nifi

Apache Nifi is very user friendly and requires no-code to very less code in some cases. While connecting the processor with other processors it requires to mention whether this is to success, failure, terminate, response and retry depends on the relation and task of the processor. In our case the first processor is connected with the second one and here we checklist the success and retry checkbox to mention to the processor that when the data is fethced and it is success then the data is passed onto the next processor to perform its task and keep on retrying for the data updates.
